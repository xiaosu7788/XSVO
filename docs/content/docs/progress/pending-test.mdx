# 生图工作台待测试

- 任务卡片现在应支持直接点击打开详情弹窗，`Ctrl / Command` 点击用于单卡增减选择，画布空白区域拖拽用于框选任务，且框选时不再选中文字。
- 失败任务卡片才会显示“重试任务”按钮，右侧铅笔按钮用于复用提示词；“加入素材 / 已加入素材”会直接反馈即时状态。
- 底部编辑区整体收窄到更接近原版的宽度，模型选择器和参数选择器也同步压窄，发送按钮图标改为白色并重新对齐。
- 编辑框支持从剪贴板直接粘贴图片参考图，且新增了展开 / 收起输入框按钮。
- Agent 模式任务卡片已改为更窄的布局，任务详情仍沿用弹窗查看图片、参数、收藏、加入素材、下载、复用和删除。
- 设置弹窗已固定为可滚动内容区，API 配置里的“XSVO 全局模型渠道”卡片已删除，选择器视觉样式继续向原版靠拢。
- 任务批量操作条已重新放回画布中部偏下的位置，和编辑区相对居中对齐。
- 本轮继续把生图工作台的编辑区、设置弹窗和任务详情弹窗收窄，并把设置里的选择器样式统一成更轻量的圆角样式。
- 设置页的 API 配置下拉已全部挂上统一的轻量样式，生图工作台里的模型/尺寸/质量等选择器也继续收窄。
- 当前用户端源码里未再找到 `API URL` / `API Key` / `模型 ID` 的输入渲染，若页面仍显示旧块大概率是缓存或未更新部署。
- 任务详情弹窗已改成更宽的双栏布局，左侧支持图片预览、多图切换、下载和加入素材，右侧补上生成总耗时、完成时间和失败信息。
- 编辑区底部工具栏已改成显式回调，避免依赖父作用域状态，提示词库 / 素材库按钮应继续保持可用。
- 旧版静态 Playground 包和宿主清缓存逻辑已删除，当前 `/image` 页面应只使用项目原生生图工作台。
- 生图工作台的系统渠道图片生成已改为后端任务轮询，并把 API 接口、流式传输、中间步骤图数量、返回 Base64、请求超时、输出格式、透明背景、审核和压缩率参数传入后端；部署环境中开启流式传输后，上游完成时本地任务应从“生成中”切到“已完成”。
- Images API / Responses API 的流式 SSE 响应现在会解析最终图片和失败事件；一次生成多张图片会拆成多个后端任务并发提交，避免第二个任务把上一个任务挂住。

> 需要人工确认：任务卡片详情弹窗是否稳定打开、设置弹窗下拉样式是否足够接近原版、剪贴板粘贴图片是否能在当前浏览器正常触发、框选后批量操作栏位置是否仍然舒服。
