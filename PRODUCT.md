# Product

## Register

product

## Users

The primary users are the project owner and a controlled set of registered users who use the app as an AI visual creation workspace. They work inside a canvas-heavy product interface where project continuity, account separation, and predictable controls matter more than marketing presentation.

## Product Purpose

XSVO is being adapted from a local-first AI canvas into a multi-user creative workbench with login, registration, role-based access, and an administrator area. Success means preserving the existing canvas, model, prompt, asset, and agent workflows while adding clear user ownership, administrative control, and a foundation for later server-side project storage.

## Brand Personality

Quiet, capable, focused. The interface should feel like a reliable creative operations tool: calm enough for long sessions, direct enough for power users, and polished without decorative distraction.

## Anti-references

Do not turn product surfaces into landing pages. Avoid oversized hero sections, decorative card grids, loud gradients, glassy dashboard effects, and promotional copy inside the app. Admin pages should not feel like a template marketplace dashboard; they should prioritize scanning, comparison, and action.

## Design Principles

- Preserve the canvas as the center of gravity.
- Add account and admin features without interrupting creative flow.
- Prefer explicit state, readable permissions, and boringly reliable controls.
- Keep admin screens dense, calm, and task-oriented.
- Build the first multi-user layer in a way that can later move local canvas data to server storage.

## Accessibility & Inclusion

Use clear labels, readable contrast, keyboard-friendly form controls, visible error states, and reduced-motion-safe interactions. No special accessibility target has been requested, but new product surfaces should be built toward WCAG AA expectations where practical.
